# Agent Loop Spec

## 背景
当前 Codia 的 `ChatService` 已有基础的"用户提问 → 模型回答（含一次工具调用）→ 返回结果"链路。但 `MAX_TOOL_ROUNDS = 1` 且每次只取第一个工具调用，模型无法自主进行多轮思考-行动-观察的循环。用户必须手动催促每一步，无法像使用 Claude Code 那样给一个任务就让 AI 自己干完。

## 目标
为 Codia 实现 Agent Loop，让 LLM 进入 ReAct 模式的自主循环：思考 → 调用工具 → 观察结果 → 继续思考/调整 → 直到任务完成。从被动应答的聊天机器人升级为能自主干活、能同时调用多个工具、能分批并发执行的 AI Agent。

同时引入 Plan Mode（`/plan` + `/do`）两段式工作流：先让模型用只读工具探索代码并输出计划到指定文件，用户确认后再放开全工具去执行。

## 功能需求
- F1: ReAct 循环 — 系统进入自主循环：将用户输入发给 LLM，解析响应中的文本和工具调用，执行工具，将结果回灌给 LLM，继续下一轮思考。循环持续直到满足任一停止条件。
- F2: 停止条件 — 至少支持以下停止条件：(a) 模型返回纯文本无工具调用（自然完成）；(b) 达到可配置的迭代上限（兜底安全网）；(c) 用户主动取消（AbortSignal）；(d) 模型本轮请求的所有工具均不存在（防止幻觉循环）；(e) 模型流式输出过程中发生网络/协议错误。
- F3: 异步事件流 — 定义一套事件类型（文本增量、工具调用开始/参数增量/完成、工具执行结果、Token 用量统计、循环轮次进度），Agent Loop 通过 AsyncIterable 向外推送事件，界面层只需消费事件即可渲染，不耦合循环逻辑。
- F4: 流式收集器 — 在接收 LLM 流式响应时，同时做两件事：实时 yield 文本 chunk 给界面（低延迟），并内部累积完整文本内容。循环结束时用完整内容判断是否含工具调用、决定下一步行为。
- F5: 多工具调用分批执行 — 当 LLM 一次返回多个工具调用时，按工具的安全性标签（readOnly/destructive）分批：(a) 所有只读工具并发执行；(b) 有副作用的工具串行执行。分批逻辑在 Agent Loop 层完成，工具本身不感知。
- F6: Plan Mode 两段式 — (a) `/plan` 阶段：通过 prompt 约束模型行为，要求其只能使用只读工具探索代码，分析需求，将执行计划写入指定的 plan file，不能做任何修改操作；(b) `/do` 阶段：恢复全部工具，模型按已确认的计划执行。若在非 plan 模式下收到 `/do` 则无操作。

## 非功能需求
- N1: 低延迟流式 — 从 LLM 返回文本 chunk 到界面渲染的延迟应尽量短，流式收集器的转发不应引入额外缓冲。
- N2: 可测试性 — Agent Loop 的核心逻辑（停止条件判断、分批策略、事件流）应可脱离真实 LLM 调用进行单元测试。循环不应硬编码在 ChatService 的私有方法里。
- N3: 可配置性 — 迭代上限应可通过配置文件设置默认值，单次调用也可覆盖。
- N4: 单工具失败不中断循环 — 一个工具执行失败时，错误结果应回灌给 LLM 让它自行调整，而不是终止整个循环。

## 不做的事
- 权限系统（工具调用前的用户确认弹窗）
- 上下文压缩（对话过长时自动裁剪/总结历史）
- 用户交互式确认（工具执行前弹出 y/n 确认）
- 工具调用的超时控制（单工具运行时间不限）
- Agent Loop 的暂停/恢复（只有取消，没有暂缓）
- 多 Agent 协作/委派
- 取消时的消息历史回滚（取消时当前轮的部分工具结果可能已写入历史，不会回滚）

## 验收标准
- AC1: 给 LLM 一个需要多步工具调用才能完成的任务（如"读取 package.json，然后在 README.md 中补充项目名称"），观察 LLM 自主完成至少 2 轮工具调用的循环，最终输出任务结果。
- AC2: 启动 Agent 后手动按 Ctrl+C 取消，观察循环立即停止，不产生新的 LLM 请求。
- AC3: 设置迭代上限为 2，给一个不可能在 2 轮内完成的任务，观察达到上限后循环终止并给出提示。
- AC4: 给 LLM 的任务需要同时读取 2 个不同文件（模型返回 2 个只读工具调用），观察它们被并发执行，总耗时接近单次读取而非两次之和。
- AC5: 给 LLM 的任务需要先写文件再读取（模型返回同时包含写和读的工具调用），观察副作用工具串行、只读工具并发的分批行为。
- AC6: 输入 `/plan` 后，模型只进行代码探索和计划输出，不对项目文件做任何修改。输入 `/do` 后恢复全部工具可用。
