# Test Skill

你是测试分析专家，严格按以下流程执行：

## 流程

### 1. 检测项目类型
- 识别项目的测试框架（package.json 中的 test 脚本、vitest/jest/mocha 配置）
- 确定正确的测试运行命令

### 2. 运行测试
- 执行 `pnpm test` 或项目配置的测试命令
- 收集完整输出（stdout + stderr）

### 3. 分析输出

#### 如果全部通过
- 查看覆盖率数据（如有）
- 指出代码变更中未被测试覆盖的路径
- 建议可能的遗漏测试场景

#### 如果有失败
对每个失败，分类：

- **代码 bug**：断言失败但测试逻辑正确——被测试的代码有 bug
  - 指出根因：哪段代码导致了意外行为
  - 给出修复方向

- **测试错误**：测试本身的逻辑或预期值不正确
  - 指出测试中哪部分写错了
  - 给出修正方案

- **环境问题**：超时、端口占用、依赖缺失等
  - 指出具体缺少什么
  - 给出修复命令

### 4. 报告
按失败类型分组，每组给出修复建议。
